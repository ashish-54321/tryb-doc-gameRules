★ The idea in one line

Today AI changes how a game looks (cosmetics). The next step: AI changes how a game plays — push speed, win distance, time limits. The same Ship Fight engine then powers a Monkey Tug of War, an Arm Wrestle, a Rocket Race… each a user-made variant created through chat.

1 Where this fits

The variant pipeline already exists. Every variant lives at /gameLibrary/{id}, carries a cosmetic blob, and its variantId already travels through matchmaking into the live match. Rule customization simply adds a second blob: rules.

2 The Ship Fight engine — what it really is

Strip away the ships and ocean and Ship Fight is a pure mechanic: two players, one position bar (0.0 – 1.0). Each tap nudges the bar toward your side. Reach the edge → you win. That's it. The "ships" are just a skin on top of this tug-of-position loop.

◄ LOSE EDGE 🐵 🐵 WIN EDGE ►
Same engine — here skinned as two monkeys pulling a rope. Tap fast → drag the cursor → push them off the edge.
Why this matters

Because the engine is abstract, any "two players fight along a line by tapping" idea is just a variant — a different skin + different rule numbers. No new engine, no new netcode. Monkey Tug of War is literally Ship Fight with a jungle skin and slightly heavier pulls.

4 Ship Fight — customizable rules

RuleWhat it doesDefaultAI range
pushPerTap How far each tap moves the bar (push / pull strength) 0.022 0.010 – 0.040
winThreshold How close to the edge counts as a win 0.04 0.02 – 0.10
maxTapRate Max taps/sec the server accepts (anti-spam) 15 8 – 25
gameDurationSec Time limit. 0 = classic sudden-death 0 0, or 15 – 60
comebackBoost Losing player gets a small push multiplier true true / false
⚠ Hard safety rule

AI never writes a raw rule value into the database. Every field passes through a server-side validator that clamps it to the allowed range. A prompt like "make me win instantly" can only ever produce winThreshold: 0.10 — the safe maximum — never something game-breaking. The server stays the single source of truth for the match.

★ The pattern

10 cards above = 10 games, 0 new engines. Each is just cosmetic (skin) + rules (numbers) on the same tap-race loop. Slow + heavy → tug-of-war feel. Fast + timed → race feel. Short edge → quick brawl. The community can invent endless concepts inside these dials.

6 Dummy JSON — "Monkey Tug of War" variant

User prompt: "a jungle tug-of-war between two monkeys — make the pulls feel slow and heavy, and you need to really drag them far to win"

// Stored at /gameLibrary/{newId} { "name": "Monkey Tug of War", "baseGameId": "ship_fight", // same engine! "creatorUid": "u_3xK9...", "isOfficial": false, "createdAt": 1715000000000, // Existing blob — looks (already shipped) "cosmetic": { "emoji": "🐵", "primaryColor": "#6B8E23", "bgTheme": "magic", // jungle-ish palette "arenaTitle": "MONKEY TUG OF WAR", "tapButtonEmoji": "🪢", // rope instead of ⚡ "collisionEmoji": "💢" }, // NEW blob — gameplay rules "rules": { "pushPerTap": 0.014, // heavy, slow pulls (clamped) "winThreshold": 0.09, // must drag them FAR "maxTapRate": 14, "gameDurationSec": 0, // no timer — pure endurance "comebackBoost": true // keeps it tense } }
How the engine uses it

Two players join "Monkey Tug of War"; the match's variantId points to this entry. The server's submitShipTap reads rules.pushPerTap (0.014) and rules.winThreshold (0.09) instead of the defaults — so each tap pulls gently and you must drag the rival 90% of the track to win. The client renders 🐵 monkeys on a 🪢 rope. Same engine, same netcode — only the numbers and skin differ.

7 User flow — making the variant

1

Open AI Assistant → Customize Game → Ship Fight

Same entry point as cosmetic customization — one chat handles both looks and rules.

2

Describe the mode in plain words

"A jungle tug-of-war between two monkeys — slow heavy pulls, drag them far to win." AI maps intent → skin + rule values.

3

AI proposes + explains the trade-off

"I'll set pushPerTap to 0.014 (heavy pulls) and winThreshold to 0.09 (long drag) — tense, endurance matches. 🐵 monkeys on a 🪢 rope, jungle palette. Add a timer?"

4

AI refuses the impossible, offers alternatives

"I can't make the rope physically stretch — the engine is a position bar. But heavy pulls + a long edge already feels exactly like a real tug-of-war."

5

Preview + Apply

Inline preview shows the skin + a rule summary ("🐌 Heavy pulls · 🎯 Long drag · ∞ No timer"). Tap Apply → forkVariant saves cosmetic + rules together.

6

"Monkey Tug of War" goes live for everyone

A new tile appears in every player's Game Library. Anyone taps it → normal matchmaking → the same engine runs with the custom skin and rules.

8 Cosmetic vs Rule customization — same pipeline

🎨 Cosmetic (done)

  • Changes how the game looks
  • emoji, colors, bg theme, SVG
  • Client-side render only
  • Zero risk to fairness

⚙️ Rules (next)

  • Changes how the game plays
  • push strength, win distance, timer
  • Server-enforced in match logic
  • Bounded — validator clamps all values

9 Why this architecture wins

♾️

Infinite content, zero new code

One engine + parameters = endless game modes. No engineer needed per variant — Monkey Tug of War shipped without touching the engine.

👥

User-generated game design

Players become game designers. Each good variant is free, organic content + a reason to return.

🛡️

Server stays authoritative

Rules are validated + clamped server-side. Anti-cheat is untouched — players can never inject values.

🔁

Reuses everything built

variantId plumbing, /gameLibrary, forkVariant, the chat UI — all already shipped. Rules = just one more blob.

📊

Built-in A/B testing

playCount per variant shows which rule sets are fun. Promote community winners to official modes.

🚀

Scales to every game

Same pattern as cosmetics — add a rules schema per engine in one shared registry file.

✓ Bottom line

Rule customization is a small, low-risk extension of an architecture already proven across 4 games. It adds one validated rules blob and one server-side read — and in return turns a single Ship Fight engine into an endlessly replayable catalogue: Monkey Tug of War, Arm Wrestle, Rocket Race, and whatever the community dreams up next.
